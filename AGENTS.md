# Laneya ÔÇô AI Agent Guide

Self-hosted YouTube subscription tracker: Rust/Axum backend, HTMX + Tailwind CSS v4 frontend, SQLite via Diesel ORM.

## Architecture

```
main Ôćĺ db::init (migrations) Ôćĺ AppState::new Ôćĺ scheduler::spawn_feed_refresh_job Ôćĺ web_server::init
```

**Layer order:** `router` Ôćĺ `handlers` Ôćĺ `repositories` Ôćĺ `models` / `services`

- **`AppState`** (`src/app_state.rs`): holds only `feed_tx: broadcast::Sender<()>` ÔÇö the single signal for "feeds changed, push new HTML to all WS clients".
- **`Config`** (`src/config.rs`): `LazyLock<Config>` singleton; always access via `Config::get()`. Reads env vars at first use.
- **`db`** (`src/db.rs`): no connection pool ÔÇö `db::establish_connection()` is called per operation; `DATABASE_URL` must be set.
- **`schema.rs`**: auto-generated by Diesel CLI ÔÇö never edit by hand.

## Request Flow

**Page routes** (`/`, `/channels`) render full Askama templates.  
**API routes** (`/api/channels/*`, `/api/videos`) return HTML fragments consumed by HTMX.  
**WebSocket** (`/ws`): on connect, sends the initial video-grid HTML; then blocks on `feed_tx.subscribe()` and pushes a re-rendered `VideoGridTemplate` whenever the scheduler detects feed changes.

## Real-time Notification Pipeline

```
scheduler (tokio::spawn, interval) 
  Ôćĺ ChannelRepository::fetch_all_feeds()
    Ôćĺ feed_reader_service::read_channels_feed()  [300 ms throttle between requests]
    Ôćĺ VideoRepository::upsert_from_feed()
  Ôćĺ if changed: ws_service::send_refresh_notification(feed_tx)  [broadcasts ()]
    Ôćĺ ws_handler receives () Ôćĺ re-renders VideoGridTemplate Ôćĺ pushes HTML over WebSocket
```

## Templates

Askama templates live in `templates/`. Each template file maps to a struct in `src/templates.rs`:

```rust
#[derive(Template)]
#[template(path = "components/video_grid.html")]
pub struct VideoGridTemplate { ... }
```

Templates use Jinja2-style syntax. `base.html` inlines HTMX and Tailwind CSS via `{% include %}` ÔÇö **no CDN**.  
Adding a new template requires: (1) create the HTML file, (2) add a struct in `src/templates.rs`, (3) render with `render_template!(MyTemplate { ... })`.

## Key Macros (`src/macros.rs`)

- `render_template!(template_expr)` ÔÇö renders an Askama template, falls back to `server_error` on failure, returns `Html<String>`.
- `html_error!("msg {}", val)` ÔÇö returns an inline `<p class="text-red-500 text-sm">ÔÇŽ</p>` for HTMX partial error display.

## Developer Workflows

**Run locally** (requires `DATABASE_URL` and optionally a `.env` file):
```sh
DATABASE_URL=./database.db cargo run
```

**Build CSS** (must be done before first run or after changing templates):
```sh
cd templates && pnpm install && pnpm run watch:css
# or one-shot:
cd templates && npx tailwindcss -i ./styles/styles.css -o ./styles/tailwind.css
```

**Add a DB migration:**
```sh
diesel migration generate <name>
# edit migrations/<timestamp>_<name>/up.sql and down.sql
diesel migration run    # applies and regenerates src/schema.rs
```

**Docker build:**
```sh
docker compose up --build
```

## Project-Specific Conventions

- **Error handling in handlers**: return `html_error!(...)` for user-facing inline errors; use `anyhow::Result` in repositories/services.
- **Feed throttle**: `read_channels_feed` inserts a 300 ms delay between requests to avoid YouTube 403 responses (see `FEED_FETCH_DELAY` in `feed_reader_service.rs`).
- **Shorts filtering**: `VideoRepository::should_filter_video` drops entries whose URL contains `/shorts/` when `FILTER_OUT_SHORTS=true`.
- **Bulk channel insert**: `ChannelRepository::bulk_create` skips existing channels by `channel_id` and returns `(inserted, skipped)`.
- **`VideoResource`** vs **`Video`**: `VideoResource` is a join query result (includes `channel_name`, human-readable `published_at`) used for rendering; `Video` is the raw DB row.
- **Tracing**: default filter is `laneya=debug,tower_http=debug`; override with `RUST_LOG` env var.
- **Socket activation**: `web_server` supports `listenfd` for systemd-style socket passing; falls back to binding `HOST` env var (`0.0.0.0:8080`).
